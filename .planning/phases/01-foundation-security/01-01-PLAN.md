a missing `.env` lets the app run with a forgeable session key, and makes all request
failures visible in production via journalctl.

server that cannot silently start insecure and whose errors are observable.

   ```python
   class JsonFormatter(logging.Formatter):
       def format(self, record: logging.LogRecord) -> str:
           payload: dict = {
               "timestamp": self.formatTime(record, "%Y-%m-%dT%H:%M:%S"),
               "level": record.levelname,
           }
           # Request-scoped fields injected via record.__dict__ by the middleware
           for field in ("method", "path", "status_code", "duration_ms"):
               if hasattr(record, field):
                   payload[field] = getattr(record, field)
           if record.exc_info:
               payload["exc"] = self.formatException(record.exc_info)
           else:
               payload["message"] = record.getMessage()
           return json.dumps(payload, ensure_ascii=False)
   ```

3. **Lifespan function** — defines startup sequence (per D-04, D-08):
   ```python
   @asynccontextmanager
   async def lifespan(app: FastAPI):
       # SEC-01: reject insecure or missing SECRET_KEY (per D-08)
       _secret = os.getenv("SECRET_KEY", "")
       _dev_default = "dev-secret-key-change-in-production"
       if not _secret or _secret == _dev_default:
           raise RuntimeError(
               "SECRET_KEY is not set or still equals the dev default. "
               "Set a strong SECRET_KEY in your .env file before starting the app."
           )

       # SEC-02: configure JSON structured logging to stdout (per D-01, D-02, D-06)
       handler = logging.StreamHandler(sys.stdout)
       handler.setFormatter(JsonFormatter())
       logging.basicConfig(level=logging.INFO, handlers=[handler], force=True)

       yield
   ```

       response = await call_next(request)
       duration_ms = round((time.perf_counter() - start) * 1000)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env → app startup | SECRET_KEY read from environment; absence or dev-default must abort startup |
| HTTP client → FastAPI | All inbound requests cross here; paths/methods logged but not trusted |
| admin route → SMTP | Invite email send is untrusted external I/O; failure must not be silent |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | session cookie / itsdangerous | mitigate | SEC-01: lifespan raises RuntimeError if SECRET_KEY missing or equals dev default — app refuses to start with forgeable keys |
| T-01-02 | Tampering | session.py _SECRET_KEY fallback | mitigate | Replace `os.getenv(..., default)` with `os.environ[...]` — KeyError if not set, no silent insecure fallback |
| T-01-03 | Information Disclosure | stdout JSON logs | accept | Logs contain method + path only (no request bodies, no PII); internal LAN deployment — journald access already restricted |
| T-01-04 | Denial of Service | lifespan RuntimeError on startup | accept | Intentional — better to fail fast and visibly than run with a broken key |
| T-01-05 | Elevation of Privilege | admin.py silent invite email failures | mitigate | SEC-02 D-05: replace `except: pass` with `log.exception(...)` so invite failures are logged and operator is alerted |
| T-01-06 | Repudiation | HTTP requests with no audit trail | mitigate | SEC-02 D-03: middleware logs one JSON line per request (method, path, status, duration) to stdout |
</threat_model>
