<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| templates instance → all routers | Shared singleton; misconfiguration (wrong directory) would break all template renders simultaneously |
| domain entity → route handlers | annual_cost() is now pure domain logic; no external input reaches it |
| ORM default callables → SQL Server | Timezone-aware datetimes written to DATETIME2 columns |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | models.py datetime defaults | mitigate | Replace bare `datetime.now` callable with `lambda: datetime.now(timezone.utc)` — prevents silent timezone drift if server moves |
| T-02-02 | Tampering | annual_cost() duplication | mitigate | Single definition in domain entity — future multiplier changes can only be made in one place; no drift risk |
| T-02-03 | Information Disclosure | passlib[bcrypt] transitive dep | mitigate | Remove passlib; pin bcrypt directly — eliminates passlib's known AttributeError with bcrypt>=4.0 which could leak tracebacks |
| T-02-04 | Denial of Service | Jinja2Templates per-router instantiation | accept | Low risk — Jinja2 caches internally; consolidation is a maintainability fix, not a security one |
| T-02-05 | Repudiation | stale DDL comment in session.py | accept | Comment removal has no security impact; reduces confusion risk only |
</threat_model>
