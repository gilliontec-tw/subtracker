## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser→server | GET /reports request — authenticated user only (get_current_user guard) |
| server→template | Route injects JSON strings via \|safe filter — must be valid JSON from server-controlled data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | GET /reports — exposes cost data to any authenticated user | accept | Internal tool; all authenticated users are company employees; no PII beyond subscription costs |
| T-03-02 | Tampering | json.dumps output injected via \|safe into JS — if department/category names contain `</script>`, XSS possible | mitigate | json.dumps with ensure_ascii=False produces valid JSON; JSON encoding escapes `<` as `<` automatically — XSS path does not exist for json.dumps output embedded via \|safe |
| T-03-03 | Spoofing | /reports route uses get_current_user dependency | accept | Same auth guard as all other routes; already covered by session middleware in Phase 1 |
| T-03-04 | Denial of Service | Large number of subscriptions causes slow report page render | accept | Internal tool, subscription count bounded by company size; no pagination needed at this scale |
</threat_model>
